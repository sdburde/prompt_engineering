02 — MODEL CONFIGURATIONS AND OUTPUT PARAMETERS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

─────────────────────────────────────────
KEYWORDS
─────────────────────────────────────────

temperature    → Controls randomness. 0 = deterministic. >1 = very random.
                 Low (0–0.3): classification, extraction, structured output.
                 High (0.7–1.2): creative writing, brainstorming.

top_p          → Nucleus sampling. Only sample from top-p probability mass.
                 top_p=0.1 → very focused. top_p=0.9 → diverse.
                 Use temperature OR top_p, not both.

max_tokens     → Hard limit on output length. Controls cost and latency.

stop           → Halt generation at a specific string.
                 stop=["4."] → model stops before item 4 in a list.

seed           → Makes outputs reproducible. Same seed + same temp = same output.

─────────────────────────────────────────
STRUCTURED OUTPUT
─────────────────────────────────────────

Ask for JSON: "Return only valid JSON, nothing else."
temperature=0 for reliable JSON output.
Always add try/except when parsing model JSON.
